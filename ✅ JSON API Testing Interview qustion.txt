✅ JSON & HTTP Methods – API Testing Interview Questions and Answers
🔶 A. JSON – Interview Questions
1. What is JSON?

JSON (JavaScript Object Notation) is a lightweight format used to send and receive data in APIs.

2. Why is JSON used in API Testing?

Easy to read and write

Lightweight

Faster parsing

Works with all programming languages

Used by almost all REST APIs

3. What are the data types supported by JSON?

String

Number

Boolean

Object

Array

Null

4. What is a JSON Object?

Data written in key-value pairs.
Example:

{"name": "John"}

5. What is a JSON Array?

Collection of values in square brackets.
Example:

["red", "blue", "green"]

6. How do you validate JSON in API testing?

Validate keys

Validate values

Validate data types

JSON Schema validation

Check nested objects and arrays

7. What is JSON Schema?

A structure that defines what a valid JSON should look like (fields, types, etc.)

8. Difference between JSON and XML?
JSON	XML
Lightweight	Heavier
Faster	Slower
Uses objects	Uses tags
Easy to read	Verbose
🔶 B. HTTP Methods – Interview Questions
1. What are HTTP methods?

Operations used to interact with REST APIs.

2. What is GET method?

Used to retrieve data from the server.
Example: Fetch all users.

3. What is POST method?

Used to create new data on the server.
Example: Create a new user.

4. What is PUT method?

Used to fully update an existing resource.

5. What is PATCH method?

Used to partially update an existing resource.

6. What is DELETE method?

Used to remove data from the server.

7. Difference between PUT and PATCH?
PUT	PATCH
Full update	Partial update
Replaces entire record	Updates specific fields
8. What is HEAD method?

Same as GET but returns only headers, not body.

9. What is OPTIONS method?

Returns supported HTTP methods for the API.

📘 2. HTTP Methods in API Testing

Operations used to interact with server using APIs.

Data retrieve करने के लिए

Body नहीं होता

Q3. POST Method
